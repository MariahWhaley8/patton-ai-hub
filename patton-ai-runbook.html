Patton AI · Runbook · Solution: Amicus

Amicus — Slack Legal Assistant

How to operate and maintain Amicus: the Slack assistant that answers staff questions from the firm's case data, notes, and Google Drive documents — and writes updates back to Base44.

Overview

What Amicus is

A Slack-native assistant, reachable by DM or @Amicus mention, that uses a hybrid retrieval design: exact case facts come from direct SQL against a synced mirror of the Base44 data; narrative and documents come from semantic (vector) search. It can update existing cases immediately and create new ones only after you confirm.

Slack  ──►  Amicus (Claude, tool-using agent)
                ├─ search_cases      → SQL over synced case data      (exact facts)
                ├─ search_documents  → Voyage + pgvector              (notes + Drive)
                ├─ find_cases / update_case / create_case → write-back to Base44
        Postgres (Railway) holds the structured mirror + vectors.

Hosting & code

Where it runs

  • Code: GitHub repo (private); app code in the legal-claude 2/ folder, branch main.
  • Host: Railway project — two services: the bot and a Postgres (with pgvector). Service root directory is legal-claude 2.
  • Deploys: Railway auto-deploys on every push to main. To change Amicus: edit code → commit → it rebuilds automatically.

Configuration

Environment variables (Railway)

Secrets live only in Railway variables — never in code. Values omitted here.

VariablePurpose
DATABASE_URLPostgres connection (auto-provided by Railway)
ANTHROPIC_API_KEY / ANTHROPIC_MODELThe model + model string
VOYAGE_API_KEY / VOYAGE_MODELEmbeddings (voyage-law-2)
SLACK_BOT_TOKEN / SLACK_APP_TOKEN / SLACK_SIGNING_SECRETSlack (Socket Mode)
SLACK_LOG_CHANNELOptional — channel id for write-back audit posts
BASE44_TOKEN / BASE44_APP_IDBase44 API (the bare key only)
GOOGLE_SERVICE_ACCOUNT_JSON / GDRIVE_FOLDER_IDDrive access (JSON key + bare folder id)
Two gotchas: BASE44_TOKEN must be only the 32-char key (not a docs snippet), and GDRIVE_FOLDER_ID must be only the id (not the full folder URL).

Slack app

  • Name/icon: Amicus + scales-of-justice mark. Socket Mode: On. Messages Tab: On (with "allow users to send messages" checked).
  • Events: app_mention, message.im.
  • Bot scopes (10): app_mentions:read, chat:write, im:history, im:read, im:write, channels:history, channels:read, groups:history, groups:read, reactions:write.
Remember: adding any scope requires Reinstall to Workspace to take effect; the sidebar name comes from App Home → Display Name.

Google Drive

  • Service account with the Drive API enabled + a JSON key.
  • The target folder must be shared with the service account's email (it only sees shared folders).
  • GDRIVE_FOLDER_ID = the bare id from the folder URL.

Day-to-day

Using Amicus

  • DM the pinned Amicus conversation, or @Amicus in any channel it's been invited to (/invite @Amicus).
  • It reacts 👀 on receipt, ✅ when it answers, ⚠️ if it errors.
  • Facts (status, balances, dates, counts) → exact SQL. Context (notes, correspondence, documents) → semantic search with citations.

Write-back

  • Update an existing case: just tell Amicus; it applies the change immediately and confirms.
  • Create a new case: Amicus asks you to confirm the details first and only creates it after a yes.
  • Audit: every write appends a timestamped note to the case in Base44 (old → new + source) and, if set, posts to the log channel.

Keep it healthy

Maintenance

Refresh data from the Railway service Console:

python -m ingest.run_ingest all       # base44 + gdrive + slack
python -m ingest.run_ingest base44     # re-sync case data
python -m ingest.run_ingest gdrive     # documents (incremental — skips indexed)
python -m ingest.run_ingest slack      # channel history
  • Re-runs are safe (upsert); the Drive sync is incremental. For long jobs run detached: nohup … &.
  • Consider a scheduled cron for base44 to keep the mirror fresh.
  • A redeploy restarts the container (kills a background ingest — just resume it).

Lessons baked in

  • Base44 uses an api_key header; store only the key.
  • Vector search casts the query embedding to ::vector; text is sanitized (NUL/control chars) before insert.
  • Each Drive file is wrapped in try/except so one bad file can't abort the batch.
  • Voyage free tier is rate-limited until a payment method is added.

Security

  • All secrets in Railway variables only; rotate any key that was ever exposed.
  • The SQL tool is read-only, single-statement; writes go only through explicit functions.
  • Limit which channels Amicus is invited to — it can read + answer from their history.
Compliance: hosting (Railway), model (Anthropic), embeddings (Voyage/MongoDB), chat (Slack), docs (Google), and the case system (Base44) are all SOC 2 Type II — see Standards & Tech.
