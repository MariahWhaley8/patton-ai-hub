How to read this

The SOC 2 rule

Tag every vendor. Not everything needs SOC 2 — here's the test.

SOC 2 audits a company that stores or processes your data. So we tag vendors, not software libraries or our own code. A tool needs SOC 2 if it stores or processes sensitive / client / regulated data. It's N/A when it's open-source software we run (the compliance boundary is the host), our own application code (it inherits the host's posture), or a tool that never touches sensitive data.

✓ SOC 2 Type II  Independently audited — cleared for sensitive/client data.
⚠ Conditional  Depends on tier or hosting — read the note before using with real data.
N/A  Open-source software or our own code — the host carries the certification.
Separate decision: whether Patton pursues its own SOC 2 report is driven by what our clients require — distinct from choosing compliant vendors.
